操作系统的概念

计算机系统的层次结构

裸机（纯硬件） -> 操作系统 -> 应用程序（软件）-> 用户

操作系统

  1. 负责管理协调硬件、软件等计算机资源的工作
  2. 为上层的应用程序、用户提供简单易用的服务
  3. 操作系统是系统软件，而不是硬件

定义 Operating System(OS) 是指控制和管理整个计算机系统的硬件和软件资源，并合理的组织调度计算机的工作和资源分配，以提供给用户和其他软件方便的接口和环境，它是计算机系统中最基本的系统软件。

操作系统的功能和目标

作为系统资源的管理者

ps: 进程 是一个程序的执行过程。执行前 需要将该程序放到内存中 ，才能被CPU处理。

  • 提供的功能
    • 文件管理 -> 打开文件夹，知道.exe文件位置
    • 存储器管理 -> 需要把该程序相关的数据放入内存
    • 处理机管理 -> 对应的进程被处理机（CPU）处理
    • 设备管理 -> 需要将摄像头等分配给进程

+目标

  • 安全高效

作为用户和计算机硬件之间的接口

  • 提供的功能

    • 命令接口 -> 允许用户直接使用

      • 联机命令接口
      • 脱机命令接口

    • 程序接口 -> 允许用户通过程序间接使用

    • GUI (图形用户界面) -> 现代操作系统中最流行的图形用户接口

  • 目标

    • 方便用户使用

联机命令接口 用户说一句，系统做一句 = 交互式命令接口
脱机命令接口 用户说一堆，系统做一堆 = 批处理命令接口
ps: .bat 文件，里面有一系列所需要做的任务清单，操作系统根据此，逐条执行 -> 脱机命令接口
程序接口 如C:\\Windos\System32\user32.dll 程序员在程序中调用 user32.dll （该过程即为 系统调用 ）即可实现创建窗口等功能，只能通过用户程序 间接使用 。
ps:系统调用 = 系统命令调用 = 广义指令
GUI(Graphical User Interface) 用户可以使用形象的图形界面进行操作

作为最接近硬件的层次

功能与目标 -> 实现对硬件机器的扩展。
没有任何软件支持的计算机称为裸机。在裸机上安装的操作系统可以提供资源管理功能和方便用户的服务功能，将裸机改造成为功能更强、使用更方便的机器
通常把覆盖了软件的机器称为扩展机，又称为 虚拟机 。

操作系统的四个特征

  • 并发
  • 共享
  • 虚拟
  • 异步

ps:并发与共享是最基本的特征，互为存在条件

并发

  • 并发 是指两个或多个事件在同一时间间隔内发生。这些事件宏观上是同时发生的，但是微观上是交替发生的。
  • 并行 是指两个或多个事件在同一时刻同时发生

  操作系统的并发性指计算机系统中同时存在着多个运行着的程序。
  一个单核处理机（CPU）同一时间只能执行一个程序，因此操作系统会负责协调多个程序交替执行（这些程序微观上交替，宏观上同时执行）
ps:当今的4核CPU意味着同一时刻可以有4个程序并行执行，但是并发也不可缺少。

共享

  • 共享 即资源共享，是指系统中的资源可供内存中多个并发执行的进程共同使用。
    • 互斥共享 -> 虽然可以提供给多个进程使用，但是同一时间内，只允许一个进程访问该资源 -> qq视频聊天
    • 同时共享 -> 允许一个时间段内由多个进程”同时”对它们进行访问 -> 同时发送文件

ps:此处 同时 往往是微观上交替，宏观上并行。（TDM）

  • 并发性指计算机系统中同时存在着多个运行着的程序
  • 共享性指系统中的资源可供内存中多个并发执行的进程共同使用

ps:失去并发性，共享性失去意义。失去共享性，无法并发。

虚拟

把一个物理上的实体变为若干逻辑上的对应物。物理实体是实际存在的，而逻辑上对应物是用户感受到的。
ps:一个程序需要放入内存并给它分配CPU才能执行。

虚拟存储技术

将运行时所需远大于运行内存的程序，可以同时运行，利用空分复用技术。

虚拟技术

实际上只有一个CPU，但是可以有多个程序同时服务，利用时分复用技术。

可以看出如果失去并发性，虚拟性也就没有存在意义。

异步

指多道程序环境下，允许多个程序并发执行，但由于资源有限，进程的执行 不是一贯到底的，而是走走停停，一不可预知的速度向前推进 ，这就是进程异步性。
ps：一个资源被一个程序调用，此时其他资源就无法调用它，必须等这个程序结束之后，才能对此资源进行调用。 -> 说明并发时，不是一贯到底的，而是走走停停。 -> 只有有了并发性，才可能导致异步性。
