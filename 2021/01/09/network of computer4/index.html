多少层次恰当

  • 太少，协议太复杂
  • 太多，描述和综合各层功能系统任务是遇到较多的困难

各层主要功能

  1. 差错控制 -> 使相应层次对等方的通信更加可靠
  2. 流量控制 -> 发送端的发送速率必须使接收端来得及接收，不要太快
  3. 分段和重组 -> 发送端将要发送的数据块划分为更小的单位，在接收端将其还原
  4. 复用和分用 -> 发送端几个高层会话复用一条低层的连接，接收端再进行分用
  5. 连接建立和释放 -> 交换数据前先建立一条逻辑连接，数据传输结束后释放连接

计算机网络的体系结构

  • architecture 是计算机网络各层及其协议的集合。
  • 体系结构就是这个计算机网络机器部件所应完成的功能精确定义

具有五层协议的体系结构

  • OSI 七层协议体系结构

  • TCP/IP 四层体系结构

    • 应用层
    • 运输层
    • 网际层
    • 网络接口层

  • 往往采用折中办法，采用一种只有 五层协议 的体系结构

OSI

  1. 应用层
  2. 表示层
  3. 会话层
  4. 运输层
  5. 网络层
  6. 数据链路层
  7. 物理层

TCP/IP体系结构

  • 应用层 -> (7.6.5) 各种应用层协议，如DNS、HTTP、SMTP等
  • 运输层 -> 4 TCP或UDP
  • 网际层 IP -> 3
  • 网络接口层 -> 1 没有具体内容

五层协议内容

  • application layer 应用层 -> 7.6.5
  • transport layer 运输层 -> 4
  • network layer 网络层 -> 3
  • data link layer 数据链路层 -> 2
  • physical layer 物理层 -> 1

ps: 运输层、网络层 重点
ps: 协议是 水平的 ，即协议是控制对等实体之间通信的规则
ps: 服务是 垂直的 ，即服务是由下层向上层通过层间接口提供的
本层服务用户只能看见服务而无法看见下面的协议。

服务访问点

  同一系统相邻两层的实体进行交互的地方称为 服务访问点 SAP(Service Access Point) 是一个抽象的概念，实际上就是一个逻辑接口。
  OSI 把层与层之间交换的数据单位称为 服务数据单元 SDU(Service Data Unit) 。

协议复杂性

  • 协议必须把所有不利的条件事先都估计到
  • 看一个协议是否正确，不光是正常情况下是否正确，还必须检查能否应对各种异常情况。

TCP/IP 体系结构

  路由器在转发分组时，最高只用到网际层，而没有使用运输层和应用层。（网络接口层与网际层）

TCP/IP协议族

  • 应用层 -> HTTP···SMTP···DNS··· RTP
  • 运输层 -> TCP与UDP
  • 网际层 -> IP
  • 网络接口层 -> 网络接口
