system call

什么是系统调用，有何作用？

操作系统提供两种服务

  • 面向用户的 命令接口 — 允许用户直接使用

    • 联机命令接口：用户说一句，系统做一句
    • 脱机命令接口：用户说一堆，系统做一堆

  • 面向程序的 程序接口：由一组系统调用组成

“系统调用”是操作系统提供给应用程序（程序员/编程人员）使用的接口，可以理解为一种可借应用程序调用的特殊函数， 应用程序可以发出系统调用请求来获得操作系统的服务。

总结：

  1. 系统调用时操作系统提供给应用程序使用的接口
  2. 应用程序通过系统调用来请求获得操作系统的服务
  3. 系统调用会使处理器从用户态进入核心态

问题： 操作系统为什么要提供”系统调用”功能？
答：打印论文并没有混在一起，而是顺序打印

问题： 如果各个进程可以随意的使用打印机会怎么样？
答： 后半段将混合在一起

解决方法：操作系统提供”系统调用”功能，用户进程想要使用打印机这种共享资源，只能通过系统调用向操作系统发出请求。操作系统会对各个请求进行协调管理。

应用程序通过系统调用请求操作系统的服务，系统中的各种共享资源都有操作系统统一掌管，因此在用户程序中，凡是与资源有关的操作（如存储分配、I/O操作、文件管理等），都必须通过系统调用的方式向操作系统提出服务请求，由操作系统代为完成。这样可以保证系统的稳定性和安全性，防止用户进行非法操作。

系统调用（按功能分类）

  • 设备管理 - 完成设备的 请求/释放/启动 等功能
  • 文件管理 - 完成文件的 读/写/创建/删除 等功能
  • 进程控制 - 完成进程的 创建/撤销/阻塞/唤醒 等功能
  • 进程通信 - 完成进程间的 消息传递/信号传递 等功能
  • 内存管理 - 完成内存的 分配/回收 等功能呢

ps:系统调用相关处理涉及到对系统资源的管理、对进程的控制。这些功能需要执行一些特权指令才能够完成，因此系统调用的相关处理需要在核心态下进行。

ps:凡是与资源有关的操作、会直接影响到其他进程的操作，一定需要操作系统介入，即需要通过系统调用来实现

系统调用和库函数的区别

  理论上应用程序可以通过汇编语言直接调用硬件系统。但是现在的程序中一般来说都是使用高级语言提供的库函数。但是这些高级语言的低层会封装一些相应的系统调用，只不过细节被库函数屏蔽了，所以使用库函数比较方便，不需要复杂的系统调用。
  系统调用的相关处理在核心态进行。

普通应用程序 编程语言 操作系统 裸机
可直接进行系统调用，也可以使用库函数。有的库函数涉及系统调用，有的不涉及 向上提供库函数。有时会将系统调用封装成库函数，以隐藏系统调用的一些细节，使上层进行系统调用更加方便 向上提供系统调用

不涉及系统调用的库函数： 如取绝对值的函数
设计系统调用的函数： 如创建一个新的文件函数

ps:系统调用屏蔽了一些对硬件的操作细节，而库函数由进一步屏蔽了一部分的系统调用细节。

总结：

  1. 系统调用是操作系统向上层提供的接口
  2. 有的库函数是对系统调用的进一步封装
  3. 当今编写的应用程序大部分是通过高级语言提供的库函数间接地进行系统调用

系统调用背后的过程

高级语言视角：调用了一行行的代码，比如说库函数write(“abc”)该库函数内部封装了系统调用的复杂细节
汇编语言视角（运行在用户态）：前期处理相关指令 -> movl指令（将”abc”作为系统调用参数放到某个通用寄存器中）-> int x 指令（trap/陷入/访管指令） -> 后续处理相关指令（运行在核心态）

ps: int 指令的参数x指明了系统调用号。此处的int不是整数的意思，其实是interrupt的缩写。不同系统有不同的系统调用号对应不同的指令。

过程：

  1. 传递系统调用参数
  2. 执行陷入指令（用户态）
  3. 执行系统调用相应服务程序（核心态）
  4. 返回用户程序

注意：

  1. 陷入指令是在用户态执行，执行陷入指令之后立刻引发一个内中断，从而CPU进入核心态
  2. 发出系统调用请求是在用户态，而对系统调用的相关处理在核心态下进行
  3. 陷入指令是唯一一个只能在用户态下执行，而不可在核心态执行的指令

ps：系统调用发生在用户态，对系统调用的处理发生在核心态、执行陷入指令会产生内中断，使处理器从用户态进入核心态。

terminals and exceptions

中断机制的诞生

  早期计算机只能串行的进行工作，同一时刻最多一道程序。为了解决这个问题，人们发明了操作系统作为计算机的管理者，引入中断机制，实现了多道程序并发执行。

本质， 发生中断意味着需要操作系统介入，并展开管理工作。

  1. 进程1正在进行，CPU收到计时部件发出的 中断信号 ，切换为 核心态 对中断进行处理
  2. 操作系统内核负责对中断信号进行处理 -> 进程1时间片已用完，换为进程2运行
  3. 进程2发出系统调用（内中断信号），请求输出。CPU切换为 核心态 ，对中断进行处理。
  4. 操作系统内核负责对中断信号进行处理
  5. 进程2要等待I/O设备运行，换进程3运行
  6. I/O完成，设备向CPU发出 中断信号
  7. CPU收到I/O设备发来的中断信号，切换为核心态，对中断进行处理
  8. 操作系统内核负责对中断信号进行处理
  9. 进程2的I/O已经完成，让进程2 恢复运行，以完成后续工作

中断的概念和作用

  1. 当中断发生时，CPU立即进入核心态
  2. 当中断发生以后，当前运行的进程暂停运行，并且由操作系统内核对中断进行处理
  3. 对于不同的中断信号，会进行不同的处理

  发生了中断，就意味着需要操作系统介入，开展管理工作。由于操作系统的管理工作（比如进程切换、分配I/O设备等）需要使用特权指令，因此CPU要从用户态转为核心态。中断可以是CPU从用户态切换为核心态，是操作系统获得计算机的控制权。有了中断，才能实现多道程序的并发执行。

ps:用户态、核心态之间的切换是怎么实现的？
答：”用户态 -> 核心态” 是通过中断实现的，并且中断是唯一途径。
“核心态 -> 用户态” 的切换是通过执行一个特权指令，将程序状态字（PSW）的标志位设置为”用户态”。

中断（广义）的分类

内中断 - （异常、例外。陷入）- 与当前程序有关

信号来源：CPU内部 与当前执行的指令有关

  • 自愿中断 — 指令中断 如：系统调用时使用的访管指令（又叫陷入指令、trap指令）
  • 强迫中断
    • 硬件故障 如：缺页
    • 软件中断 如：整数除0

外中断 - 中断（狭义）- 与当前程序无关

信号来源：CPU外部
与当前执行程序无关

  • 外设请求 如：I/O操作完成发出的中断信号
  • 人工干预 如：用户强行终止一个进程

另一种分类方式

  • 内中断

    • 陷阱、陷入（trap） – 有意而为的异常，如系统调用
    • 故障 （flaut） – 由错误条件引起的，可能被故障处理程序修复，如缺页
    • 终止 （abort） – 不可修复的致命错误造成的结果，终止处理程序不再将控制返回给引发终止的应用程序，如整数除0

  • 外中断

    • I/O中断
    • 人工干预

外中断的处理过程

  1. 执行完每个指令之后，CPU都要检查当前是否有外部中断信号
  2. 如果检测到外部中断信号，则需要保护中断进程的CPU环境（如程序状态字PSW、程序计数器PC、各种通用寄存器） – 详见 计算机组成原理
  3. 根据中断信号类型转入相应的中断处理程序 – 属于核心态
  4. 恢复原进程的CPU环境并退出中断，返回原进程继续往下执行

Operating System day2

操作系统的发展与分类

ps:理解各个阶段优缺点

手工操作阶段

  • 没有操作系统
  • 计算机处理速度快
  • 输入/输出速度较慢

主要缺点：

  1. 用户独占全机
  2. 人机速度矛盾导致资源利用率极低

批处理阶段

单道批处理系统

  引入脱机输入/输出技术（磁带完成），并监督程序负责控制作业的输入、输出 -> 操作系统雏形

  • 磁带速度比纸袋机快很多
  • 纸带 -> 外围机 -> 磁带 -> 处理器 -> ···

主要优点：

  1. 缓解了一定程度的人机速度矛盾
  2. 资源利用率有所提升

主要缺点：

  1. 内存中仅能够有一道程序运行，只有该程序运行结束之后才能调入下一道程序
  2. CPU有大量的时间是在空闲等待I/O完成
  3. 资源利用率依然很低

多道批处理系统 （操作系统开始出现）

  依然是磁带，每次往内存中输入多道程序。操作系统正式诞生，并引入 中断技术 ，由操作系统负责管理这些程序的运行。各阶段并发执行。

主要优点:

  1. 多道程序并发执行
  2. 共享计算机资源
  3. 资源利用率大幅度提升
  4. CPU和其他资源保持”忙碌”状态，系统吞吐量增大

主要缺点:

  1. 用户响应时间长
  2. 没有人机交互功能 （用户提交自己的作业之后就只能等待计算机处理完成，中间不能控制自己的作业执行）

分时操作系统

  计算机以时间片为单位轮流为各个用户/作业服务，各个用户可通过终端与计算机进行交互。

主要优点:

  1. 用户请求可以被即时响应，解决了人机交互问题。
  2. 允许多个用户同时使用一台计算机
  3. 用户对计算机的操作相互独立，感受不到别人的存在

主要缺点:

  1. 不能优先处理一些紧急任务，操作系统对各个用户/作业都是完全公平的
  2. 循环地为每个用户/作业服务一个时间片，不区分任务的紧急性

实时操作系统

主要优点:

  1. 能够优先响应一些紧急任务，某些紧急任务不需时间片排队

  在实时操作系统的控制下，计算机系统接收外部信号后及时进行处理，并且要在严格的时限内处理完事件。实时操作系统的主要特点是 及时性和可靠性 。

  • 硬实时系统 必须在绝对严格的规定时间内完成处理 — 导弹控制系统 、 自动驾驶系统
  • 软实时系统 能接受偶尔违反时间规定 — 火车订票系统

网络操作系统

实现网络中各种资源的共享和各台计算机之间的通信

分布式操作系统

  • 分布性和并行性
  • 任何工作都可以分布在这些计算机上，由它们并行、协同完成这个任务

个人计算机操作系统

OS的运行机制和体系结构

运行机制

两种指令

  • 什么是指令
    • 一条高级语言的代码翻译过来可能会对应多条指令
    • 指令指处理器CPU能够识别、执行的最基本命令（二进制语言）
    • 有的程序有很高的权限 如内存数据清零
      • 特权指令 不允许用户程序使用
      • 非特权指令

两种处理器状态

  用户状态字寄存器（PSW）中的某标志位来识别当前处理器处于什么状态。如0为用户态、1为核心态

  • 核心态 目态

    • CPU只能执行非特权指令

  • 用户态 管态

    • 特权指令、非特权指令都可以执行

两种程序

  • 内核程序

    • 操作系统的内核程序是系统的管理者，既可以执行特权指令，也可以执行非特权指令，运行在核心态

  • 应用程序

    • 为了保证系统能安全运行，普通应用程序只能执行非特权指令、运行在用户态

操作系统内核

  那些属于内核程序？

  • 离纯硬件最近的部分

    1. 时钟管理

      • 实现计时功能。

    2. 中断处理

    3. 原语

      • 原语是一种特殊的程序
      • 是最接近硬件的部分，中程序的运行具有原子性。
      • 运行时间较短、调用频繁

ps:这种程序要么不执行，要么执行就必须一次执行到底，不可被打断

  • 此外也是内核 对系统资源进行管理的功能
    1. 进程管理
    2. 存储器管理
    3. 设备管理

ps:有的操作系统不把这部分功能归为”内核功能”。也就是说不同的操作系统，对内核功能的划分可能并不一样

  • 大内核 - 包含的部分比较多

    • 将操作系统的主要功能模块都作为系统内核，运行在核心态

    • 优点

      1. 高性能

    • 缺点

      1. 内核代码庞大
      2. 结构混乱
      3. 难以维护

  • 微内核 - 包含的部分比较少

    • 只把最基本的功能保留在内核

    • 优点

      1. 内核功能少
      2. 结构清晰
      3. 方便维护

    • 缺点

      1. 需要频繁地在核心态和用户态之间切换
      2. 性能低

ps:内核是计算机上配置的低层软件，是操作系统最基本、最核心的部分。
ps:实现操作系统内核的那些程序就是内核程序。

HTML 常见基础标签

标签 – tag

Tag <html> <head> <meta>
分为单标签 <meta> 双标签<html> <head> <body> <title>

  1. 标签是HTML语言基本组成部分
  2. 标签是分层的；多标签总是成对出现，包括开始标签和结束标签
  3. 根标签是html，包括两个子标签head 和 body
  4. 标签与属性不区分大小写 如 <html> <HTML> 都可以

属性与属性值 – attribute

  1. 附属于标签，对标签的特征进行设置
  2. 属性的值可以用双引号引起来 （单引号也可以）
  3. 属性和属性值不区分大小写
  4. 属性之间无先后次序，属性也可以省略 -> 默认
  5. 属性都是写在开始标签中

多少层次恰当

  • 太少，协议太复杂
  • 太多，描述和综合各层功能系统任务是遇到较多的困难

各层主要功能

  1. 差错控制 -> 使相应层次对等方的通信更加可靠
  2. 流量控制 -> 发送端的发送速率必须使接收端来得及接收，不要太快
  3. 分段和重组 -> 发送端将要发送的数据块划分为更小的单位，在接收端将其还原
  4. 复用和分用 -> 发送端几个高层会话复用一条低层的连接，接收端再进行分用
  5. 连接建立和释放 -> 交换数据前先建立一条逻辑连接，数据传输结束后释放连接

计算机网络的体系结构

  • architecture 是计算机网络各层及其协议的集合。
  • 体系结构就是这个计算机网络机器部件所应完成的功能精确定义

具有五层协议的体系结构

  • OSI 七层协议体系结构

  • TCP/IP 四层体系结构

    • 应用层
    • 运输层
    • 网际层
    • 网络接口层

  • 往往采用折中办法，采用一种只有 五层协议 的体系结构

OSI

  1. 应用层
  2. 表示层
  3. 会话层
  4. 运输层
  5. 网络层
  6. 数据链路层
  7. 物理层

TCP/IP体系结构

  • 应用层 -> (7.6.5) 各种应用层协议，如DNS、HTTP、SMTP等
  • 运输层 -> 4 TCP或UDP
  • 网际层 IP -> 3
  • 网络接口层 -> 1 没有具体内容

五层协议内容

  • application layer 应用层 -> 7.6.5
  • transport layer 运输层 -> 4
  • network layer 网络层 -> 3
  • data link layer 数据链路层 -> 2
  • physical layer 物理层 -> 1

ps: 运输层、网络层 重点
ps: 协议是 水平的 ，即协议是控制对等实体之间通信的规则
ps: 服务是 垂直的 ，即服务是由下层向上层通过层间接口提供的
本层服务用户只能看见服务而无法看见下面的协议。

服务访问点

  同一系统相邻两层的实体进行交互的地方称为 服务访问点 SAP(Service Access Point) 是一个抽象的概念，实际上就是一个逻辑接口。
  OSI 把层与层之间交换的数据单位称为 服务数据单元 SDU(Service Data Unit) 。

协议复杂性

  • 协议必须把所有不利的条件事先都估计到
  • 看一个协议是否正确，不光是正常情况下是否正确，还必须检查能否应对各种异常情况。

TCP/IP 体系结构

  路由器在转发分组时，最高只用到网际层，而没有使用运输层和应用层。（网络接口层与网际层）

TCP/IP协议族

  • 应用层 -> HTTP···SMTP···DNS··· RTP
  • 运输层 -> TCP与UDP
  • 网际层 -> IP

Operating System day1

日常操作系统

  • Windows
  • android
  • IOS
  • Mac OS
  • Linux

知识总览

  • 操作系统的
    • 概念
    • 功能和目标
      • 资源的管理者 -> 安全、高效
      • 向用户提供服务 -> 使用方便
      • 对硬件机器的扩展 -> 扩展能力

操作系统的概念

计算机系统的层次结构

裸机（纯硬件） -> 操作系统 -> 应用程序（软件）-> 用户

操作系统

  1. 负责管理协调硬件、软件等计算机资源的工作
  2. 为上层的应用程序、用户提供简单易用的服务
  3. 操作系统是系统软件，而不是硬件

定义 Operating System(OS) 是指控制和管理整个计算机系统的硬件和软件资源，并合理的组织调度计算机的工作和资源分配，以提供给用户和其他软件方便的接口和环境，它是计算机系统中最基本的系统软件。

操作系统的功能和目标

作为系统资源的管理者

ps: 进程 是一个程序的执行过程。执行前 需要将该程序放到内存中 ，才能被CPU处理。

  • 提供的功能
    • 文件管理 -> 打开文件夹，知道.exe文件位置
    • 存储器管理 -> 需要把该程序相关的数据放入内存
    • 处理机管理 -> 对应的进程被处理机（CPU）处理
    • 设备管理 -> 需要将摄像头等分配给进程

+目标

  • 安全高效

作为用户和计算机硬件之间的接口

  • 提供的功能

    • 命令接口 -> 允许用户直接使用

      • 联机命令接口
      • 脱机命令接口

    • 程序接口 -> 允许用户通过程序间接使用

    • GUI (图形用户界面) -> 现代操作系统中最流行的图形用户接口

  • 目标

    • 方便用户使用

联机命令接口 用户说一句，系统做一句 = 交互式命令接口
脱机命令接口 用户说一堆，系统做一堆 = 批处理命令接口
ps: .bat 文件，里面有一系列所需要做的任务清单，操作系统根据此，逐条执行 -> 脱机命令接口
程序接口 如C:\\Windos\System32\user32.dll 程序员在程序中调用 user32.dll （该过程即为 系统调用 ）即可实现创建窗口等功能，只能通过用户程序 间接使用 。
ps:系统调用 = 系统命令调用 = 广义指令
GUI(Graphical User Interface) 用户可以使用形象的图形界面进行操作

作为最接近硬件的层次

功能与目标 -> 实现对硬件机器的扩展。
没有任何软件支持的计算机称为裸机。在裸机上安装的操作系统可以提供资源管理功能和方便用户的服务功能，将裸机改造成为功能更强、使用更方便的机器
通常把覆盖了软件的机器称为扩展机，又称为 虚拟机 。

操作系统的四个特征

  • 并发
  • 共享
  • 虚拟
  • 异步

ps:并发与共享是最基本的特征，互为存在条件

并发

  • 并发 是指两个或多个事件在同一时间间隔内发生。这些事件宏观上是同时发生的，但是微观上是交替发生的。
  • 并行 是指两个或多个事件在同一时刻同时发生

  操作系统的并发性指计算机系统中同时存在着多个运行着的程序。
  一个单核处理机（CPU）同一时间只能执行一个程序，因此操作系统会负责协调多个程序交替执行（这些程序微观上交替，宏观上同时执行）
ps:当今的4核CPU意味着同一时刻可以有4个程序并行执行，但是并发也不可缺少。

共享

  • 共享 即资源共享，是指系统中的资源可供内存中多个并发执行的进程共同使用。
    • 互斥共享 -> 虽然可以提供给多个进程使用，但是同一时间内，只允许一个进程访问该资源 -> qq视频聊天
    • 同时共享 -> 允许一个时间段内由多个进程”同时”对它们进行访问 -> 同时发送文件

ps:此处 同时 往往是微观上交替，宏观上并行。（TDM）

  • 并发性指计算机系统中同时存在着多个运行着的程序
  • 共享性指系统中的资源可供内存中多个并发执行的进程共同使用

ps:失去并发性，共享性失去意义。失去共享性，无法并发。

虚拟

把一个物理上的实体变为若干逻辑上的对应物。物理实体是实际存在的，而逻辑上对应物是用户感受到的。
ps:一个程序需要放入内存并给它分配CPU才能执行。

虚拟存储技术

将运行时所需远大于运行内存的程序，可以同时运行，利用空分复用技术。

虚拟技术

实际上只有一个CPU，但是可以有多个程序同时服务，利用时分复用技术。

可以看出如果失去并发性，虚拟性也就没有存在意义。

异步

指多道程序环境下，允许多个程序并发执行，但由于资源有限，进程的执行 不是一贯到底的，而是走走停停，一不可预知的速度向前推进 ，这就是进程异步性。
ps：一个资源被一个程序调用，此时其他资源就无法调用它，必须等这个程序结束之后，才能对此资源进行调用。 -> 说明并发时，不是一贯到底的，而是走走停停。 -> 只有有了并发性，才可能导致异步性。
