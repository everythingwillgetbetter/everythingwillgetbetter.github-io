行书

主要笔法

  笔画中牵丝是承上启下地路过，不是笔画，为此，通常牵丝不应该下重笔，不宜写得明显粗重。牵丝运用要顺其自然，不要太多，要善于表现笔断意连得承启效果。
  行书笔画主要分八中：

  • 运笔。即指用笔书写笔画时的起笔、行笔、收笔这三个连贯动作。

    • 其左端为逆锋起笔，左右两端之间为行笔。右端为藏锋收笔。

  • 提、按。即运笔过程中的轻提与重按。

    • 提笔使笔尖触纸的力度较轻，笔画则细；按笔使笔尖触纸的力度重，其笔画则粗。

  • 翻、折。翻笔，是指向上翻抽笔尖；折，通常指写折时笔尖向右、向下的走势。

为什么会有乱码

  • 数据存储在硬盘，看不见
  • 可以看见分区C、D、E盘
  • 硬盘上有许多小磁铁，分为N极与S极，存储数据时，就是改变这些小磁铁的磁极，将其区分为N极与S极。及代表0/1–最终存储为磁盘上的0/1。
  • 必须遵守存储规则–字符集（字典）：代表文字转换对应关系。
  • 字符集不止一个，如GBK(国标，专门处理中文)与UTF-8（既有中文、又有国际通用字符）

怎么处理乱码

  • 弄清存储的字符集是哪个

    • 另存为 -> ANSI（本地默认字符集）

  • 告诉浏览器用的编码 meta – 单标签，有开头无结尾，多一个斜线

  • 处理打开问题 -> 运行配置 -> 编辑 -> 找浏览器启动路径 -> 找浏览器.exe文件位置

计算机网络类别

计算机网络定义

 &emxp;计算机网络主要是有一些通用的、可编程的硬件互联而成，而这些硬件并非专门用来实现某一特定目的。这些可编程的硬件能够用来传送多种不同类型的数据，并能够支持广泛的和日益增长的应用。

分类

  • 从网络的作用范围进行分类

    • 广域网 WAN（Wide Area Network）–几十到几千公里
    • 城域网 MAN （Metropolitan Area Network）–5~50公里
    • 局域网 LAN （Local Area Network）–1公里
    • 个人区域网 PAN（Personal Area Network）–10米

  • 从网络的使用者进行分类

    • 公用网 (public network)
    • 专用网 （private network）

  • 用来把用户接入到互联网的网络

    • 接入网 AN （Access Network）本地接入网/居民接入网
    • 不属于互联网核心，也不属于边缘部分
    • 起到用户与互联网间的桥梁作用

计算机网络的性能

性能指标

  1. 速率

    • 单位 bit/s
    • 指的是数据 传送速率 ，也称为 data rate 或 bit rate
    • b -> kb -> Mb （1024）
    • b/s -> kb/s -> Mb/s (1000)

  2. 带宽

    • bandwidth，原指信号具有的频带宽度，其单位是Hz
    • 计算机网络中，带宽指用来表示网络中某通道传送数据能力，即单位时间内网络中某信道所能通过的 最高数据率 。
    • 单位 bit/s

ps:时间轴上信号的宽度随带宽的增大而变小

  1. 吞吐量
    • 表示单位时间内通过某个网络（或信道、接口）的数据量
    • 以便知道实际上到底有多少数据量能够通过网络
    • 受网络的带宽或网络的额定速率限制

ps:网络的好坏不是看单个因素，单个因素提高到一定程度，会陷入瓶颈，其他因素成为制约的主要问题。

  1. 时延 （delay或latency）
    • 指数据从网络的一段传送到另一端所需的时间
    • 迟延、延迟
    • 由发送时延、传播时延、处理时延、排队时延组成

ps:发送时延 = 数据帧长度 / 发送速率
ps:传输时延 = 信道长度 / 信号在信道上的传播速度
ps:排队时延往往取决于网络中当时的通信量

  1. 时延带宽积

    • 单位为 bit
    • 时延带宽积 = 传播时延 * 带宽
    • 并不是说越大越好还是越小越好，只能说在带宽确定时 越小越好 ，或者时延确定时 越大越好 。
    • 只有在代表链路的管道都充满bit时，链路才得到了充分利用

  2. 往返时间 RTT

    • 互联网不仅仅是单向传输，而是一次交互，需要知道双向交互一次所需要的时间
    • 表示从发送方发送数据开始，到发送方收到来自接收方的确认，所需要的时间

  3. 利用率

    • 分为信道利用率与网络利用率
    • 信道利用率指出某信道有百分之几的时间是被利用过的。完全空闲的信道的利用率是零。
    • 网络利用率则是全网络的信道利用率加权平均值。
    • 信道利用率并非越到越好，利用率增大时，该信道引起的时延也就迅速增加。

ps:有点像门限效应
ps:时延与网络利用率关系

  • 当某信道的利用率增加是，该信道引起的时延也就迅速增加
  • 若令D0表示网络空闲时延，D表示网络当前时延，则在适当的假设条件下，可以利用

D = D0/1-U
ps：其中U是网络的利用率，0-1之间

非性能指标

  • 费用
  • 质量
  • 标准化
  • 可靠性
  • 可扩展性和可升级性
  • 易于管理与维护

计算机网络体系结构的形成

  相互通信的两个计算机系统必须 高度协调工作 才行， 分层 可将庞大而复杂的问题转化为若干较小的局部问题。

开放系统互联参考模型 OSI/RM –（Open Systems Interconnection Reference Model）

法律上的国际标准OSI（de jure）没有得到市场的认可，但是非国际标准TCP/IP（de facto）却获得了最广泛的应用。

协议与划分层次

  网络中数据交换必须遵守的实现约定规则，明确了规定所交换的数据格式以及有关同步问题。
  网络协议（network protocol）简称协议，是为进行网络中数据交换而建立的规则、标准或约定。

  • 语法 – 格式
  • 语义 – 响应
  • 同步 – 顺序

ps:表面上是两个主机之间直接通信，实际上，是交由下层模块进行发送处理的。

好处 缺点
各层之间是独立的 降低效率
灵活性好 有些功能会在不同的层次中重复出现，因而产生了额外开销
结构上可分割开
易于实现和维护
能过促进标准化工作

网络的网络

  • 计算机网络 （网络）
  • 互联网（网络的网络）

网络中 node-结点 数据结构tree中 node-节点

  • 云两种情况
    • 云已经包含了与网络相连的计算机
    • 云只包含路由器以及和连接这些路由器的链路，把有关的计算机画在云的外面

网络把许多计算机连接在一起，互联网则把许多网络通过路由器连接在一起。

  • ISP（Internet Service Provider）
  • www(World Wide Web)

互联网标准化工作

互联网协会ISOC

  • 互联网研究部 IRTF
  • 互联网工程部 IETF

互联网组成

  • 互联网边缘部分： 由所有连接互联网的主机组成，用户直接使用，通信与资源共享
  • 互联网核心部分：由大量网络和连接这些网络的路由器组成，提供连通性和交换

边缘部分的主机又称其为端系统（end system）->端到端通信

所谓通信：运行在主机A上的某个程序和运行在主机B上的另一个程序进行通信

  • 客户服务器方式
  • 对等连接方式（peer to peer简称P2P）指两个主机通信的时候并不区分哪个是服务请求方还是服务提供方，两个主机都运行了P2P软件才可，双方都可以下载对方已经存储在硬盘中的文档
